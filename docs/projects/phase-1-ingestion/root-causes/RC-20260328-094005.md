# RC-20260328-094005 - Forestry lean derivation drift from JS key ordering

- **Title:** Forestry lean derivation drift from JS key ordering
- **Date:** 2026-03-28
- **Owners:** Fred E
- **Linked invariant / plan section:** `docs/projects/phase-1-ingestion/ARTICLE6_INGEST_UPGRADE_PLAN.md`

## Summary
`idempotency (forestry)` and `stage-gates` failed because `node scripts/derive-lean-from-rich.js` rewrote forestry `sections.json` files even when no semantic data changed.

## Root Cause
The JS lean-derivation path wrote object keys in insertion order, while the repo's canonical lean generation expectations already relied on sorted key order. Forestry `sections.json` files include `anchor`, `section_number`, and `stable_id`, so the ordering mismatch produced deterministic byte drift during gate runs.

## Fix
Updated `scripts/derive-lean-from-rich.js` to sort object keys recursively before writing JSON so the JS path matches the canonical shell path and no longer rewrites forestry lean outputs on rerun.
