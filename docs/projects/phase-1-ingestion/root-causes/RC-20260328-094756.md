# RC-20260328-094756 - Agriculture previous META sections hash drift from stale committed audit hashes

- **Title:** Agriculture previous META sections hash drift from stale committed audit hashes
- **Date:** 2026-03-28
- **Owners:** Fred E
- **Linked invariant / plan section:** `docs/projects/phase-1-ingestion/ARTICLE6_INGEST_UPGRADE_PLAN.md`

## Summary
`idempotency (agriculture)` and `stage-gates` failed because the scoped agriculture ingest path refreshed historical `META.audit_hashes.sections_json_sha256` values for existing previous-version packs under `ACM0010`, `AMS-III.D`, and `AMS-III.R`.

## Root Cause
The committed `sections.json` bytes were already stable, but the corresponding historical `META.json` files still carried older `sections_json_sha256` values. During the gate run, `hash-all.sh` recalculated hashes from the current `sections.json` files and rewrote only `META.json`, which made CI fail the clean-tree check.

## Fix
Refresh and commit the affected historical agriculture `META.audit_hashes.sections_json_sha256` values so they match the current on-disk `sections.json` bytes. No lean or rich methodology content change is required for this fix.
