# RC-20260508-031647 — VM0007 quality standard missed rich-to-lean canonicalization path
- Date: 2026-05-08
- Area: CI / rules / determinism
Tags: [ci, determinism, schema, forestry]

## Symptom
`Devcontainer Build` and `stage-gates / gate` failed on PR #403 after `review_contract_v1` was added for `Verra/AFOLU/VM0007/v1-8`.

Representative failures:
- `AssertionError [ERR_ASSERTION]: Verra/AFOLU/VM0007/v1-8 rules.json rule R-1-0001 drifts from canonical lean contract`
- `Lean JSON drift detected. Run: node scripts/derive-lean-from-rich.js`

## Impact
The repo stored truthful draft lean artifacts for VM0007, but CI regenerated a different `rules.json` from `rules.rich.json`. That made the methodology appear non-deterministic and blocked the PR even though the committed lean file itself was valid.

## Root cause
The new `review_contract_v1` quality standard was only wired into the lean-from-lean canonicalization path. The legacy rich-to-lean path used by `scripts/derive-lean-from-rich.js` still emitted the old key order and dropped the new draft-only lean fields such as `quality_status` and empty `when`.

## Fix
- Updated `canonicalizeLeanRuleFromLegacyRich()` to respect `review_contract_v1`.
- Emitted the quality-standard key order for lean rules when the method opts in.
- Preserved explicit draft lean fields for opt-in methods, including `quality_status` and `when: []`.

## New invariants / guardrails
- If a methodology opts into a richer artifact contract, lean regeneration from rich and lean validation from committed artifacts must produce byte-identical canonical lean output.
- CI drift checks must treat explicit pending draft fields as part of the deterministic lean contract, not optional noise.

## Proof / tests
- `node scripts/derive-lean-from-rich.js`
- `bash scripts/check-lean-drift.sh`
- `node tests/methodology-artifact-contract.test.js`
- `node scripts/validate-offline.js`

## Follow-ups
- [ ] If more methodologies adopt `review_contract_v1`, keep both canonicalization paths covered by the shared artifact-contract tests.
