# RC-20260508-014340 — VM0007 draft artifacts overstated coverage
- Date: 2026-05-08
- Area: forestry
Tags: [forestry, schema, ci, tools]

## Symptom
VM0007 v1.8 had verified source provenance, but existing sections and rules could still be interpreted as completed methodology coverage even though VF2 and VF3 had not been source-audited.

## Root cause
The repo distinguished source provenance from methodology encoding only in roadmap phase text, not in machine-readable metadata consumed by validators or downstream tooling.

## Fix
Marked the verified source PDF, retained sections, and retained rules separately in VM0007 META metadata, declared external Verra dependencies as external/unencoded, added a proof test to block review-ready state while artifacts remain draft, and updated the Verra roadmap/status text.

## Proof / tests
node scripts/validate-offline.js; node tests/methodology-artifact-contract.test.js; node tests/verra-vm0007-draft-truthfulness.test.js; bash scripts/ci-run-tests.sh

## Follow-ups
- [ ] Clear the draft status only after VF2/VF3 audit confirms the retained section and rule artifacts against the verified VM0007 v1.8 source PDF.
