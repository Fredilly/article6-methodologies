# RC-20260414-060135 — Canonical lean artifact drift
- Date: 2026-04-14
- Area: schema
Tags: [schema, determinism, ci, agriculture, forestry]

## Symptom
Methods and previous releases emitted different lean rules.json and sections.json shapes, duplicate rule text, and missing canonical stable section references.

## Root cause
The shared rich-to-lean generator preserved legacy method-specific shapes, while lean schemas stayed permissive enough that structural drift across methods and releases did not fail validation.

## Fix
Centralized canonical lean normalization in the shared derive-lean-from-rich path, tightened lean schemas, declared rich overlay exceptions explicitly, and added a repo-wide artifact contract test to fail on drift.

## Proof / tests
node tests/methodology-artifact-contract.test.js; ./scripts/run-ajv.sh validate -s schemas/sections.schema.json -d 'methodologies/**/sections.json'; ./scripts/run-ajv.sh validate -s schemas/rules.schema.json -d 'methodologies/**/rules.json'

## Follow-ups
- [ ] Canonicalize rich sections and legacy rich rule overlays further if downstream consumers can drop legacy rich mode.
