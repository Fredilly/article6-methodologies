# RC-20260507-135220 — Previous-version META audit hash drift
- Date: 2026-05-07
- Area: CI / previous / META
Tags: [ci, determinism, previous, agriculture, forestry]

## Symptom
The agriculture and forestry idempotency checks, plus the stage-gates workflow, failed because scoped ingest rewrote archived META audit hash fields under methodologies slash previous.

## Impact
The branch could not pass idempotency or stage-gate CI even though rule and section payload validation succeeded.

## Root cause
The branch included the lean-contract refactor and previous-version path fixes, but the checked-in archived META audit hashes still reflected older lean bytes. Scoped ingest re-derived previous-version lean artifacts and immediately refreshed those stale hash values, so CI detected drift on the first pass.

## Fix
Refresh the affected archived META audit hashes for Agriculture and Forestry previous-version outputs so the committed values already match the deterministic lean bytes emitted by scoped ingest.

## New invariants / guardrails
Archived previous-version META audit hashes for rules and sections must be regenerated whenever canonical lean derivation changes, and they must remain clean after the first scoped idempotency run.

## Proof / tests
Failed Actions logs for runs 25499604458 and 25499604396, plus isolated local reruns of the agriculture and forestry idempotency scripts and the quality-gates check.

## Follow-ups
- [ ] Keep archived previous-version META hash refreshes coupled to future lean-contract changes.
