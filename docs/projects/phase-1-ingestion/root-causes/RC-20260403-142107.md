# RC-20260403-142107 — AR-ACM0003 RC-S5 lineage allowlist gap and RC-S6 expected-evidence proving support
- Date: 2026-04-03
- Area: forestry
Tags: [forestry, determinism, schema, ci]

## Symptom
The `AR-ACM0003` proving method did not emit RC-S5 version lineage and diff hints in current `META.json`, and its rich rules had no RC-S6 expected-evidence hints despite already carrying grounded monitoring and reporting detail.

## Impact
Requirement-coverage consumers could not navigate from current `AR-ACM0003` to its archived predecessor or consume methodology-side evidence expectations for the proving case without downstream guesswork.

## Root cause
The RC-S5 enrichment path only looked at concurrently canonical method directories, so `AR-ACM0003` lost its archived `previous/v01-0` predecessor when computing lineage. Separately, the forestry rich rewrite path had no optional expected-evidence hook for grounded rule-level evidence hints.

## Fix
Merge archived `previous/*` versions into the proving-family lineage used for RC-S5 enrichment, enable the `AR-ACM0003` family for version/diff emission, and add additive rich-only expected-evidence generation for the grounded `AR-ACM0003` monitoring, uncertainty, and reporting rules.

## New invariants / guardrails
- Previous-version-aware current `META.json` lineage must include archived predecessors recorded under `methodologies/**/previous/**`.
- RC-S5 diff hints for a current methodology version are methodology-side outputs and must not depend on downstream app transforms.
- RC-S6 expected-evidence hints remain rich-only and are omitted unless the canonical rule summary, notes, or timing text safely grounds them.
