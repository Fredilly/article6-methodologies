# RC-20260414-064630 — json canonicalizer drifted lean contract order
- Date: 2026-04-14
- Area: canonical-json / scoped ingest
Tags: [ci, determinism, schema, agriculture, forestry]

## Symptom
`stage-gates` and sector idempotency jobs failed after scoped ingest regenerated methodology artifacts, with
`tests/methodology-artifact-contract.test.js` reporting that lean `sections.json` objects drifted from the
canonical contract.

## Impact
The shared scoped-ingest path could rewrite canonical lean methodology artifacts into a structurally different
JSON shape even when the underlying data was unchanged. This broke deterministic regeneration and forced app- and
CI-facing consumers to see alphabetized methodology records instead of the canonical contract order.

## Root cause
`scripts/json-canonical-check.sh` applied a generic deep alphabetical key sort to every JSON file. Scoped ingest
ran that fixer after generation, so lean methodology artifacts (`rules.json`, `sections.json`) were rewritten from
their contract order into alphabetical order. The script also classified files using raw paths, so `/tmp` versus
`/private/tmp` symlink differences could bypass any methodology-specific handling.

## Fix
- Taught `scripts/json-canonical-check.sh` to detect methodology lean artifacts and stringify them through the
  shared lean contract helpers instead of generic alphabetical sorting.
- Normalized the checker's root and file paths through `realpath` so methodology files are classified
  consistently across local worktrees and CI workspaces.
- Added a regression test covering JSON canonicalization of lean methodology artifacts.
- Refreshed stale archived `META.json` audit hashes for `UNFCCC/Forestry/AR-AM0014/v03-0/previous/{v01-0,v02-0}`.

## New invariants / guardrails
- Canonical JSON fixing must never reorder lean methodology keys away from the shared lean artifact contract.
- Path normalization used by contract-aware tooling must be stable across symlinked workspace roots.
- Archived previous-version `META.json` hashes must match the current bytes of their checked-in lean artifacts.
