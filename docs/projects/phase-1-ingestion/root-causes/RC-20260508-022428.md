# RC-20260508-022428 — VM0007 draft artifacts lacked explicit quality shape
- Date: 2026-05-08
- Area: schema
Tags: [schema, forestry, ci, determinism]

## Symptom
VM0007 draft sections and rules mixed high-detail entries with sparse placeholders, so downstream tooling could not tell whether missing locators, source spans, or expected evidence were intentionally pending or simply absent.

## Root cause
The repo had a lean-vs-rich artifact split, but no reusable quality contract requiring explicit placeholder fields before a methodology could be considered review-ready.

## Fix
Introduced the opt-in review_contract_v1 artifact quality standard, expanded methodology schemas and contract helpers to preserve its lean field shape, normalized VM0007 v1.8 draft artifacts to that shape with explicit pending statuses, and added proof tests that guard review-readiness.

## Proof / tests
node scripts/validate-offline.js; node tests/methodology-artifact-contract.test.js; node tests/methodology-artifact-quality-standard.test.js; node tests/verra-vm0007-draft-truthfulness.test.js

## Follow-ups
- [ ] Migrate additional methodologies to review_contract_v1 only when their seed artifacts can truthfully expose explicit pending enrichment states without implying audit completion.
