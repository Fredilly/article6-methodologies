# RC-20260331-235111 — RC4 upstream monitoring-report canon ownership
- Date: 2026-03-31
- Area: schema
Tags: [schema, determinism, ci, agriculture]

## Symptom
Monitoring-report expectation fields were being introduced downstream instead of in the canonical methodologies repo.

## Impact
Canonical methodology-contract ownership drifted away from this repo, which risked incompatible app-local rich fields and source-of-truth divergence for future evidence-intake phases.

## Root cause
Repo-boundary drift let app-phase work define methodology-contract-shaped fields outside the canonical rich schema and generator path owned here.

## Fix
Add an additive rich-only monitoring_report contract under requirement_coverage.expected_evidence, generate it only for grounded AM0073 report-bound evidence, document repo ownership, and prove scoped idempotency.

## New invariants / guardrails
- Canonical methodology schema and generated artifact contract changes originate in `article6-methodologies`.
- Downstream app and evidence-intake phases consume upstream rich canon fields instead of defining methodology canon locally.
- Optional monitoring-report expectations are omitted unless the canonical source text grounds them for a specific rich expected-evidence item.
