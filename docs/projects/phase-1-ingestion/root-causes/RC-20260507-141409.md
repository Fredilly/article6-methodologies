# RC-20260507-141409 — Offline standalone validators still required ajv runtime
- Date: 2026-05-07
- Area: offline validation / standalone validators
Tags: [ci, determinism, schema]

## Symptom
node scripts/validate-offline.js failed on the intended offline path because committed standalone validators fell back when rules.cjs and sections.cjs required ajv runtime helpers.

## Impact
PR 401 could pass online validation with node_modules present but violated the repository contract that offline validation must remain dependency-free and reproducible.

## Root cause
The standalone validator generation step emitted require calls to ajv runtime helpers for Unicode string length checks, so the committed CJS validators were not actually self-contained.

## Fix
Sanitize generated standalone validator code to inline the Unicode length helper, regenerate the committed validator artifacts, and rebuild the offline bundle from the dependency-free outputs.

## New invariants / guardrails
Standalone validators under scripts slash validators must execute without node_modules, and emitted validator files may not require ajv runtime helpers or any external package at runtime.

## Proof / tests
node scripts/validate-offline.js from a sparse checkout without node_modules, node -e require scripts slash validators slash bundle.cjs, and the PR offline plus idempotency test commands.

## Follow-ups
- [ ] Keep validator generation changes coupled to offline validation reruns and bundle regeneration.
